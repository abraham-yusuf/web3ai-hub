# Liquidity Mining dan Mekanismenya

## Apa Itu Liquidity Mining?

Liquidity mining adalah mekanisme distribusi token di mana protocol memberikan reward token kepada pengguna yang menyediakan likuiditas atau berpartisipasi aktif. Ini adalah cara utama DeFi protocol **bootstrapping** likuiditas dan membangun komunitas di tahap awal.

Bayangkan sebuah pasar baru yang butuh pedagang. Pemilik pasar berkata: "Siapa yang mau berjualan di sini, saya kasih hadiah setiap hari." Itu esensi liquidity mining.

## Bagaimana Token Distribution Bekerja

Protocol menyiapkan alokasi token reward untuk didistribusikan ke pengguna berdasarkan kontribusi mereka.

```
Total Supply Token ABC: 1.000.000.000
Alokasi Liquidity Mining: 40% (400.000.000)
Distribusi: 4 tahun, dengan penurunan bertahap

Tahun 1: 150.000.000 token (37,5%)
Tahun 2: 100.000.000 token (25%)
Tahun 3: 80.000.000 token (20%)
Tahun 4: 70.000.000 token (17,5%)
```

**Reward per blok:**
```
Reward/blok = Total Alokasi / Total Blok dalam Periode

Contoh (Ethereum, ~6.500 blok/hari):
Tahun 1: 150.000.000 / (365 × 6.500) ≈ 63 token/blok
```

Pengguna mendapat reward proporsional terhadap **share** mereka di pool.

```
Reward user = (LP user / Total LP) × Reward per blok
```

## Gauge Voting: Komunitas Menentukan Arah

Beberapa protocol (terutama Curve) menggunakan **gauge voting** untuk menentukan distribusi reward ke pool mana.

```
Cara kerja:
1. Setiap pool punya "gauge" (pengukur)
2. Pemegang ve-token voting ke gauge pool favorit
3. Pool dengan vote lebih banyak → dapat lebih banyak reward
4. Voting dilakukan setiap minggu/bulan
```

**Mengapa ini penting?**
- Memberi kontrol ke komunitas
- Mencegah protocol "buang-buang" token ke pool yang tidak perlu
- Menciptakan "Curve Wars" — kompetisi untuk mendapatkan voting power

## ve-Tokenomics: Vote-Escrowed Token

**ve-token** (vote-escrowed token) adalah inovasi tokenomics yang populer oleh Curve Finance. Konsepnya: lock token lebih lama = dapat lebih banyak voting power.

```
Lock CRV → dapat veCRV

Lock 1 tahun → 0,25 veCRV per CRV
Lock 2 tahun → 0,50 veCRV per CRV
Lock 3 tahun → 0,75 veCRV per CRV
Lock 4 tahun → 1,00 veCRV per CRV (maksimum)

veCRV menurun seiring waktu → harus lock lagi untuk maintain power
```

**Manfaat ve-token:**
1. **Voting power** untuk gauge voting
2. **Boost** reward hingga 2,5x di pool
3. **Fee sharing** dari protocol
4. **Mengurangi sell pressure** (token terkunci = tidak dijual)

**Masalah ve-tokenomics:**
- Likuiditas token berkurang
- Konsentrasi power di whale besar
- Kompleksitas untuk user biasa

## Bootstrapping Liquidity

Bootstrapping likuiditas adalah tantangan "ayam dan telur" di DeFi: protocol butuh likuiditas untuk menarik user, tapi butuh user untuk mendapat likuiditas.

### Strategi Bootstrapping

**1. Liquidity Mining Launch**
- Berikan reward sangat tinggi di awal
- Tarik farmer yang mencari yield
- Reward menurun seiring waktu

```
Minggu 1-4:  APY 1000%+ (menarik perhatian)
Bulan 2-6:   APY 100-300% (stabilisasi)
Bulan 6+:    APY 10-50% (sustainable)
```

**2. Liquidity Bootstrapping Pool (LBP)**
- Harga awal tinggi, turun secara bertahap
- Membeli di awal = rugi, mendorong pembelian di harga wajar
- Menghindari sniper bot yang membeli di menit pertama

**3. Airdrop ke Liquidity Provider**
- Berikan token gratis ke LP dari protocol lain
- Contoh: Uniswap memberi UNI ke user awal
- Membangun user base yang sudah aktif

**4. Partnership dan Protocol Owned Liquidity (POL)**
- Protocol sendiri menyediakan likuiditas
- Bekerja sama dengan protocol lain untuk shared liquidity
- Menggunakan treasury untuk bootstrap

## Incentive Alignment yang Baik

Liquidity mining yang baik menciptakan **aligned incentives** antara protocol dan user:

**Buruk:**
- Reward tinggi tapi token tidak ada utility
- User datang hanya untuk reward, langsung jual
- "Farm and dump" cycle yang menghancurkan harga

**Baik:**
- Reward terkait dengan utilitas nyata (fee, governance)
- Mekanisme lock yang memberi benefit nyata
- Sustainable yield dari revenue protocol, bukan hanya token printing

## Contoh Nyata: Curve Finance

```
Curve (CRV) Liquidity Mining Ecosystem:

1. User tambah likuiditas di Curve pool
2. User stake LP token → dapat CRV reward
3. User lock CRV → dapat veCRV
4. veCRV holder voting ke gauge
5. Pool dengan gauge lebih tinggi → lebih banyak CRV
6. veCRV holder dapat boost 2,5x + fee sharing

Hasil: Ekosistem yang saling terkait dan self-reinforcing
```

"Curve Wars" terjadi ketika protocol seperti Convex dan Yearn bersaing mengumpulkan veCRV untuk mendapatkan voting power dan boost bagi user mereka.

## Evaluasi Program Liquidity Mining

Sebelum berpartisipasi, tanyakan:

1. **Sumber yield**: dari revenue atau token printing?
2. **Tokenomics**: ada supply cap? burn mechanism?
3. **Vesting**: reward langsung atau terkunci?
4. **Team allocation**: berapa persen untuk tim?
5. **Competitive moat**: apa yang membuat protocol bertahan setelah reward habis?

## Ringkasan

Liquidity mining adalah mekanisme kunci untuk bootstrapping dan mempertahankan likuiditas di DeFi. Dengan memahami gauge voting, ve-tokenomics, dan strategi bootstrapping, kamu bisa mengevaluasi program liquidity mining secara kritis. Di halaman berikutnya, kita akan membahas secara matematis salah satu risiko terbesar menjadi liquidity provider: impermanent loss.
