# DeFi Risks dan Mitigasi

## Mengapa Memahami Risiko Itu Penting?

DeFi menawarkan peluang finansial yang luar biasa — tanpa izin, tanpa perantara, 24/7. Tapi peluang ini datang dengan risiko yang tidak ada di keuangan tradisional. Tidak ada FDIC insurance, tidak ada customer service, dan tidak ada yang bisa membatalkan transaksi.

Dari tahun 2020 hingga 2025, lebih dari **$10 miliar** aset kripto hilang karena exploit DeFi. Memahami risiko adalah langkah pertama untuk melindungi diri.

## 1. Smart Contract Risk

Smart contract adalah kode yang mengontrol dana kamu. Jika ada bug, dana bisa hilang selamanya.

**Jenis kerentanan:**

```
- Reentrancy: penyerang menarik dana berkali-kali sebelum saldo diperbarui
- Integer overflow/underflow: perhitungan matematika error
- Access control: siapa pun bisa menjalankan fungsi admin
- Logic error: kode tidak melakukan yang seharusnya
```

**Contoh nyata:**
- **The DAO Hack (2016)**: $60 juta hilang karena reentrancy
- **Cream Finance (2021)**: $130 juta karena flash loan exploit
- **Euler Finance (2023)**: $197 juta karena logic vulnerability

**Mitigasi:**
- Hanya gunakan protocol yang sudah **di-audit** oleh firma terpercaya (Trail of Bits, OpenZeppelin, Consensys Diligence)
- Cek berapa lama protocol sudah live tanpa insiden
- Periksa apakah ada **bug bounty** program (hadiah untuk menemukan bug)
- Gunakan protocol dengan TVL tinggi — insentif untuk menjaga keamanan lebih besar
- Diversifikasi: jangan taruh semua dana di satu protocol

## 2. Oracle Manipulation

Oracle adalah jembatan antara data dunia nyata (harga) dan smart contract. Jika oracle dimanipulasi, seluruh protocol bisa dieksploitasi.

```
Serangan Oracle Manipulation:
1. Penyerang manipulasi harga di DEX (menggunakan flash loan)
2. Oracle melaporkan harga palsu ke lending protocol
3. Penyerang meminjam aset dengan jaminan yang terlalu murah
4. Penyerang mengantongi keuntungan
```

**Contoh:** Mango Markets ($110 juta, 2022) — manipulasi harga MNG token untuk meminjam lebih dari jaminan.

**Jenis Oracle:**
- **On-chain** (TWAP DEX): harga rata-rata dari beberapa blok, sulit dimanipulasi dalam satu transaksi
- **Off-chain** (Chainlink): data dari node independen, terpusat tapi reliable
- **Hybrid**: kombinasi keduanya

**Mitigasi:**
- Pilih protocol yang menggunakan oracle terpercaya (Chainlink, Pyth)
- Hindari protocol dengan oracle sederhana (hanya satu sumber harga)
- Waspada protocol yang menggunakan harga spot langsung dari satu DEX

## 3. Governance Attacks

Banyak DeFi protocol dikontrol oleh pemegang token governance. Jika seseorang mengumpulkan cukup token, mereka bisa mengubah aturan protocol untuk keuntungan sendiri.

```
Contoh Governance Attack:
1. Penyerang pinjam banyak token governance (flash loan)
2. Submit proposal untuk mengubah parameter
3. Vote "ya" dengan kekuatan besar
4. Eksekusi proposal untuk mengeksploitasi protocol
5. Kembalikan token pinjaman
```

**Serangan terkenal:**
- **Beanstalk (2022)**: $182 juta — flash loan governance attack
- **Build Finance (2022)**: governance takeover sederhana

**Mitigasi:**
- Protocol dengan **time-lock** (perubahan butuh waktu sebelum dieksekusi)
- Voting period yang cukup panjang (3-7 hari)
- Quorum minimum yang tinggi
- Tim yang memegang cukup token untuk memveto proposal berbahaya
- Komunitas yang aktif memantau governance

## 4. Risiko Lainnya

### Rug Pull
Developer menghilang dengan dana pengguna.
```
Tanda rug pull:
- Tim anonim
- Liquidity tidak dikunci
- Tokenomics sangat menguntungkan tim
- Tidak ada audit
- Hype berlebihan tanpa substance
```

### Centralization Risk
Protocol yang "terdesentralisasi" sebenarnya dikontrol oleh sedikit pihak.
- Multi-sig wallet dengan 3 dari 5 tanda tangan
- Admin key yang bisa mengubah semua parameter
- Single point of failure

### Regulatory Risk
- Pemerintah bisa melarang atau membatasi DeFi
- Protocol bisa memblokir wallet tertentu (USDC/USDT bisa di-freeze)
- Konsekuensi pajak yang tidak jelas

### Composability Risk (Risk Lego)
DeFi disebut "money legos" — protocol saling tergantung. Jika satu blok gagal, seluruh rantai bisa runtuh.
```
Contoh:
- Kamu stake ETH di Lido → dapat stETH
- Kamu jadikan stETH jaminan di Aave
- Kamu pinjam USDC dari Aave
- Jika Lido bermasalah → stETH jatuh → liquidation cascade
```

### Liquidity Risk
- Tidak bisa menjual aset jika tidak ada pembeli
- Pool bisa kosong jika LP menarik likuiditas massal
- Bank run di protocol lending

## 5. Asuransi DeFi

Asuransi DeFi melindungi dari kerugian akibat exploit smart contract.

```
Cara kerja:
1. Kamu beli polis asuransi (misal 2,5% dari dana per tahun)
2. Jika terjadi exploit → klaim ganti rugi
3. Klaim dievaluasi oleh DAO atau penilai independen

Protokol asuransi:
- Nexus Mutual — asuransi smart contract terbesar
- InsurAce — multi-chain coverage
- Unslashed Finance — modular insurance
```

**Kekurangan asuransi DeFi:**
- Tidak semua kerugian ter-cover
- Proses klaim bisa lambat dan kontroversial
- Premium bisa mahal untuk protocol populer
- Asuransi sendiri punya smart contract risk

## Framework Evaluasi Risiko

Sebelum menggunakan protocol DeFi, gunakan checklist ini:

```
Smart Contract:
□ Sudah di-audit oleh firma terpercaya?
□ Sudah berjalan berapa lama?
□ Ada bug bounty program?
□ TVL cukup besar?

Oracle:
□ Menggunakan oracle apa?
□ Apakah harga bisa dimanipulasi dalam satu transaksi?
□ Ada mekanisme perlindungan?

Governance:
□ Ada time-lock untuk perubahan?
□ Berapa lama voting period?
□ Siapa yang memegang voting power?

Tim:
□ Tim public atau anonim?
□ Track record sebelumnya?
□ Ada insentif untuk bertahan jangka panjang?

Ekonomi:
□ Yield dari mana? (sustainable vs token printing)
□ Ada lock period?
□ Bagaimana cara keluar?
```

## Prinsip Keamanan DeFi

1. **DYOR** (Do Your Own Research) — jangan percaya hype
2. **Diversifikasi** — jangan taruh semua di satu protocol
3. **Mulai kecil** — test dengan jumlah kecil dulu
4. **Monitor aktif** — cek posisi secara berkala
5. **Tetap teredukasi** — ikuti berita keamanan DeFi
6. **Hardware wallet** — simpan dana utama di cold storage
7. **Revoke approval** — cabut izin smart contract yang tidak dipakai

## Ringkasan

DeFi membuka peluang finansial baru, tapi datang dengan risiko yang harus dipahami. Smart contract risk, oracle manipulation, dan governance attack adalah tiga risiko terbesar. Dengan menggunakan framework evaluasi, diversifikasi, dan asuransi DeFi, kamu bisa memitigasi sebagian besar risiko. Ingat: di DeFi, kamu adalah bank sendiri — dan itu berarti kamu juga bertanggung jawab penuh atas keamanan dana sendiri.
