# DAO: Governance Terdesentralisasi yang Perlu Dipahami

DAO (Decentralized Autonomous Organization) adalah organisasi yang dioperasikan melalui kode komputer dan aturan yang ditentukan oleh smart contract, di mana keputusan dibuat melalui voting oleh anggota yang memiliki token governance.

## Sejarah Singkat DAO

- **2016** — The DAO dibaut di Ethereum, kemudian di-hack (US$60 juta)
- **2016-2020** — Belajar dari kegagalan, protokol baru muncul
- **2021+** — DeFi Summer membawa governance token mainstream
- **2025** — DAO menjadi standard untuk protocol governance

## Cara Kerja DAO

```
Token Holder
    │
    ├─── Submit Proposal (ide/gagasan)
    │
    ├─── Vote (FOR / AGAINST / ABSTAIN)
    │
    └─── Execute if threshold met
            │
            └─── Smart Contract auto-executes
```

## Komponen Utama DAO

### 1. Governance Token
Token yang memberikan hak voting. Contoh:
- UNI (Uniswap)
- AAVE (Aave)
- MKR (MakerDAO)
- COMP (Compound)

### 2. Proposal System
- **Temperature Check** — Informal signal dari komunitas
- **Governance Proposal** — Resmi, membutuhkan quorum
- **Fast Track** — Untuk emergency decisions

### 3. Governor Bravo (Compound Style)
Standard governance framework yang banyak diadopsi:
- Timelock delays
- Quorum requirements
- Proposal thresholds

## Kelebihan DAO

- **Transparency** — Semua voting dan decisions di-chain
- **Censorship resistant** — Tidak bisa dihentikan oleh satu entitas
- **Global access** — Siapa pun dengan internet dapat berpartisipasi
- **Direct incentive alignment** — Token holders aligned dengan protocol success

## Tantangan DAO

1. **Low voter participation** — Biasanya <10% dari token holder voting
2. **Token concentration** — Whales dapat mendominasi voting
3. **Coordination overhead** — Proses decision making lebih lambat
4. **Legal uncertainty** — Status hukum DAO belum jelas di banyak yurisdiksi

DAO merepresentasikan shift dari hierarchical organizations ke programmable, transparent, dan permissionless structures. Это будущее управления организациями.
