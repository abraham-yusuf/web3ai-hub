# Persiapan Airdrop Hunting: Portfolio dan Checklist Lengkap

Airdrop hunting adalah aktivitas mencari proyek kripto yang berpotensi memberikan free token rewards. Dengan preparation yang tepat, kamu bisa maximize potential rewards sambil minimizing risk.

## Wallet Setup Strategy

### Single vs Multiple Wallets

```
Wallet Hierarchy:
├── CEX (Binance/Coinbase) ─── Purchase ETH
│
├── Main Wallet (Cold/Hardware) ─── Long-term storage
│   └── 2-3 sub-wallets untuk different purposes
│       ├── Wallet A: High-value DeFi positions
│       ├── Wallet B: Active trading/speculation  
│       └── Wallet C: Airdrop hunting (clean history)
│
└── Burner Wallets ─── Testnets, new protocols
    └── High-risk, small amounts
```

## Activity Patterns

### Natural Looking Activity

```
Week 1-2: Light activity (just exploring)
Week 3-4: Regular usage (swap, lend, bridge occasionally)
Month 2: Consistent but not excessive
Month 3: Vary interactions, try new features
```

### What To Do (Legitimate Activity)

```typescript
// Do:
- Use protocols naturally
- Interact dengan different features
- Hold positions for meaningful periods
- Provide liquidity (small amounts)
- Vote in governance (if applicable)
- Bridge assets between chains
- Test withdrawals/deposits

// Don't:
- Wash trade (creates fake volume)
- Transact with yourself repeatedly
- Create hundreds of wallets (sybil detection)
- Claim all at once (looks bot-like)
- Flash loans for activity (detectable)
```

## Airdrop Checklist

### Pre-Activity

- [ ] Hardware wallet ready
- [ ] Multiple wallets created (3-5 optimal)
- [ ] ETH distributed across wallets
- [ ] RPC endpoints configured per chain
- [ ] Block explorer bookmarks ready

### Per Protocol

- [ ] Read documentation
- [ ] Understand tokenomics
- [ ] Check if testnet available
- [ ] Identify all interaction points
- [ ] Execute transactions spread over time
- [ ] Document wallet addresses used
- [ ] Save transaction hashes

### Monthly Review

- [ ] Review all protocols interacted
- [ ] Check for new mainnet deployments
- [ ] Update wallet balances
- [ ] Verify claim eligibility
- [ ] Monitor announcement channels

### Common Mistakes to Avoid

1. **Over-diversification** — Lebih baik depth di beberapa protokol daripada thin spread di banyak
2. **Ignoring gas costs** — Pastikan rewards > costs
3. **Rushing at end** — Projects can detect sudden activity spikes
4. **Same IP across wallets** — Use different RPC/configurations
5. **Forgetting to claim** — Set reminders untuk claim deadlines
